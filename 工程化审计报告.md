## 二、死代码 / 冗余代码

### 2.1 `app/src/data/knowledge.js` 约 300 行死代码

连接后端真实 AI 后，本地降级逻辑已不再被调用。外部文件只 import 以下 4 个导出：
```
brands, categories, getBrand, getCategory
```

以下导出**无任何外部引用**（仅文件内部互相调用），属于死代码：
- `articles` 数组（12 条静态装备数据，约 250 行）
- `faqs` 数组（5 条 FAQ）
- `ask()` 函数（本地匹配降级，Chat.vue 改用 WS 后不再调用）
- `getArticle()` / `searchArticles()` / `articlesByBrand()` / `articlesByCategory()`（仅被 `ask()` 内部调用）

> 建议：确认 Chat.vue 不再需要本地降级后，删除上述死代码，文件可从 561 行瘦身到约 80 行。

### 2.2 前端 4 个函数完全重复

`app/src/api/chat.js` 和 `app/src/api/product.js` 各自定义了完全相同的 4 个函数：
```
buildSpecList(specs)    // 规格列表构造
balanceLabel(value)     // 平衡点中文映射
shaftLabel(value)       // 中杆硬度中文映射
widthLabel(value)       // 鞋楦宽度中文映射
```

> 建议：抽到 `app/src/utils/specs.js` 公共模块，两边 import。

### 2.3 冗余依赖

**后端 `server/requirements.txt`**：

## 三、安全问题（按严重度）

### P0 — 必须修复（生产化阻塞项）

| # | 问题 | 位置 | 风险 |
|---|---|---|---|
| 1 | **密码用 MD5** | `server/utils/security.py:14` | MD5 已被破解，彩虹表秒破。应用 `bcrypt` 或 `argon2` |
| 2 | **JWT SECRET_KEY 硬编码默认值** | `server/config.py:20` `"badminton-rag-secret-key-2026"` | .env 未配 SECRET_KEY，用默认值。仓库 PUBLIC 后任何人可伪造 admin token |
| 3 | **CORS `allow_origins=["*"]`** | `server/main.py:40` | 生产环境不应开放所有源，应限定为前端域名 |
| 4 | **种子 admin 弱口令** | `server/bootstrap.py:77` `admin/admin` | 公开仓库后默认口令可被直接登录 |

### P1 — 应该修复

| # | 问题 | 位置 | 风险 |
|---|---|---|---|
| 5 | **无 rate limiting** | 全局 | 登录/注册/AI 接口无频率限制，易被刷（暴力破解、LLM 成本攻击）|
| 6 | **WebSocket token 走 query 参数** | `server/routers/chat.py:65` | query 会进 nginx/uvicorn 访问日志，token 易泄露。应改用首条消息或 Sec-WebSocket-Protocol |
| 7 | **上传文件仅靠 content_type 校验** | `server/routers/upload.py:29` | content_type 可伪造，应校验文件头魔术字节 |
| 8 | **`openai>=2.26.0` 版本不锁定** | `server/requirements.txt:17` | 用 `>=` 而非 `==`，构建不可复现 |

### P2 — 建议优化

| # | 问题 | 位置 |
|---|---|---|
| 9 | 密码修改后未失效旧 token | `admin_profile.py` / `user_profile.py` | 改密后旧 token 仍有效至过期 |
| 10 | 注册接口无验证码 | `auth.py:71` | 机器人可批量注册 guest 账号 |
